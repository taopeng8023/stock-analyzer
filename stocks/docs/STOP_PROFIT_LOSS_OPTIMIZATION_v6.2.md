# 🎯 止盈止损优化方案 v6.2

**优化日期：** 2026-03-21  
**优化依据：** 六轮回测数据分析  
**目标：** 止盈/止损比从 0.76:1 → 1.2:1+

---

## 📊 问题分析

### 六轮回测止盈止损统计

| 轮次 | 止盈次数 | 止损次数 | 止盈/止损比 |
|------|---------|---------|-----------|
| 第一轮 (v5.1) | 29 | 28 | 1.04:1 ✅ |
| 第二轮 (v5.1) | 32 | 36 | 0.89:1 ⚠️ |
| 第三轮 (v5.1) | 32 | 35 | 0.91:1 ⚠️ |
| 第六轮 (v6.1-ML) | 25 | 33 | **0.76:1** ❌ |

**问题：** 第六轮止盈/止损比仅 0.76:1，显著低于第一轮 1.04:1

### 原因分析

1. **止盈阈值过紧**
   - 当前止盈：+8% ~ +18%
   - 实际牛股涨幅常超 30%+
   - 过早止盈，错失大行情

2. **止损阈值过宽**
   - 当前止损：-6% ~ -12%
   - 亏损扩大后才止损
   - 单笔亏损过大

3. **缺乏移动止盈**
   - 固定止盈无法锁定利润
   - 盈利回吐严重

4. **缺乏时间止损**
   - 持仓时间过长
   - 资金效率低

## ✅ v6.2 优化方案

### 1. 进一步放宽止盈

| 波动率 | v5.1/v6.1 | **v6.2 优化** | 放宽幅度 |
|--------|----------|-------------|---------|
| 高 (>10%) | +8% | **+12%** | +50% |
| 中 (5-10%) | +12% | **+18%** | +50% |
| 低 (0-5%) | +15% | **+22%** | +47% |
| 负 (<0%) | +18% | **+25%** | +39% |

**效果：** 让利润充分奔跑，捕捉大牛股

### 2. 适度收紧止损

| 波动率 | v5.1/v6.1 | **v6.2 优化** | 收紧幅度 |
|--------|----------|-------------|---------|
| 高 (>10%) | -6% | **-5%** | -17% |
| 中 (5-10%) | -8% | **-7%** | -13% |
| 低 (0-5%) | -10% | **-8%** | -20% |
| 负 (<0%) | -12% | **-10%** | -17% |

**效果：** 快速止损，控制单笔亏损

### 3. 移动止盈 (Trailing Stop)

```python
TRAILING_STOP_CONFIG = {
    'enabled': True,              # 启用移动止盈
    'min_profit_to_activate': 0.10,  # 盈利达到 10% 后启用
    'update_frequency': 'daily',  # 每日更新
    'lock_profit': True,          # 锁定已实现盈利
}
```

**工作原理：**
1. 盈利达到 10% 后启用移动止盈
2. 股价每创新高，止盈价上移
3. 股价回撤 5-8% 时自动止盈
4. 锁定已有利润，防止回吐

**示例：**
```
买入价：10 元
止盈价：12.2 元 (+22%)

股价涨到 15 元 → 止盈价上移到 13.8 元 (15 * 0.92)
股价涨到 18 元 → 止盈价上移到 16.56 元 (18 * 0.92)
股价跌到 16.50 元 → 触发止盈，盈利 +65%
```

### 4. 时间止损 (新增)

```python
TIME_STOP_CONFIG = {
    'enabled': True,              # 启用时间止损
    'max_holding_days': 10,       # 最长持有 10 天
    'break_even_days': 5,         # 5 天后保本出局
    'reduce_position_days': 7,    # 7 天后减仓 50%
}
```

**工作原理：**
1. 持仓 5 天后，若未盈利，保本出局
2. 持仓 7 天后，若未止盈，减仓 50%
3. 持仓 10 天后，强制清仓

**效果：** 提高资金效率，避免长期套牢

### 5. 分级止盈 (新增，可选)

```python
TIERED_PROFIT_CONFIG = {
    'enabled': True,              # 启用分级止盈
    'tiers': [
        {'target': 0.10, 'sell_pct': 0.30},  # 涨 10% 卖出 30%
        {'target': 0.20, 'sell_pct': 0.30},  # 涨 20% 再卖出 30%
        {'target': 0.30, 'sell_pct': 0.40},  # 涨 30% 清仓
    ]
}
```

**效果：** 分批止盈，平衡收益与风险

### 2. 移动止盈实现

```python
def update_trailing_stop(current_price, highest_price, trailing_pct):
    """更新移动止盈价"""
    trailing_stop_price = highest_price * (1 - trailing_pct)
    return trailing_stop_price

def check_stop(current_price, stop_profit, stop_loss, 
               trailing_stop_price, trailing_enabled):
    """检查是否触发止盈止损"""
    if trailing_enabled and current_price < trailing_stop_price:
        return 'trailing_stop'
    if current_price >= stop_profit:
        return 'take_profit'
    if current_price <= stop_loss:
        return 'stop_loss'
    return 'hold'
```

### 3. 时间止损实现

```python
def check_time_stop(hold_days, current_return, config):
    """检查时间止损"""
    if hold_days >= config['max_holding_days']:
        return 'force_close'
    if hold_days >= config['break_even_days'] and current_return <= 0:
        return 'break_even'
    if hold_days >= config['reduce_position_days']:
        return 'reduce_half'
    return 'hold'
```

## ⚠️ 风险提示

1. **过度放宽止盈风险**
   - 可能错失止盈机会
   - 盈利回吐增加
   - **对策：** 启用移动止盈锁定利润

2. **过度收紧止损风险**
   - 可能被正常波动震出
   - 错过后续上涨
   - **对策：** 根据波动率动态调整

3. **移动止盈延迟风险**
   - 极端行情可能无法及时止盈
   - **对策：** 设置最低止盈保障
