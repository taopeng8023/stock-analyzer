
**问题**：涨跌幅数据异常（65%、165%、323%），严重违反真实数据原则！

---

## 🔍 根本原因

### 原因 1：腾讯 API 字段索引错误

**错误代码**：
```python
'change_pct': float(fields[39]) if fields[39] else 0,  # ❌ 这是市盈率！
```

**腾讯 API 真实字段**：
```
[3] 当前价
[4] 昨收
[31] 涨跌额
[32] 涨跌幅%  ← 应该用这个
[39] 市盈率   ← 错误地用了这个
```

### 原因 2：使用了错误的字段

- `fields[39]` = 市盈率（65.67, 324.79 等）
- `fields[32]` = 涨跌幅%（-9.13, -1.30 等）

代码错误地将市盈率当作涨跌幅使用！

---

## ✅ 修复方案

### 修复 1：更正字段索引

**修复后代码**：
```python
'change': float(fields[31]) if fields[31] else 0,      # 涨跌额
'change_pct': float(fields[32]) if fields[32] else 0,  # 涨跌幅% ✅
```

### 修复 2：添加数据验证

```python
# 真实市场数据
current_price = float(fields[3]) if fields[3] else 0
last_close = float(fields[4]) if fields[4] else 0

# 确保数据合理性
if current_price <= 0 or current_price > 10000:
    return None  # 异常数据丢弃
```

### 修复 3：添加数据来源标记

```python
'source': 'tencent',
'crawl_time': datetime.now().isoformat(),
```

确保所有数据可追溯来源。
