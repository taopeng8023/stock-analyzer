## 🔍 异常推送机制详解

### 异常层次结构

```
WorkflowException (基类)
│
├── DataFetchException (Layer 1)
│   ├── 数据源不足
│   ├── 格式验证失败
│   └── API 调用异常
│
├── AnalysisException (Layer 2)
│   ├── 输入验证失败
│   ├── 输出验证失败
│   └── 分析处理异常
│
└── PushException (Layer 3)
    ├── 输入验证失败
    ├── 输出验证失败
    └── 推送 API 异常
```

### 推送触发流程

```
Layer 异常
  ↓
捕获 WorkflowException
  ↓
提取异常信息 (layer, message, details)
  ↓
调用 PushLayer.push_error(exception)
  ↓
格式化错误报告 (Markdown)
  ↓
调用企业微信 API
  ↓
推送结果 (success/failed)
```

### 错误报告格式

```markdown
🚨 v8.0-Financial-Enhanced 基准版 - 执行异常报告
⏰{时间}
📊 仅主板 | 排除创业/科创/北交所
⚠️ 仅供参考，不构成投资建议

━━❌ 工作流执行失败━━

📋 异常信息:
  Layer: {Layer 名称}
  错误：{错误消息}
  详情：{详细信息}

💡 建议:
  - 检查数据源状态
  - 查看完整日志
  - 等待系统恢复

==================================================
_📊 v8.0-Financial-Enhanced 基准版 - 严格数据契约_
_🔒 数据格式验证失败 = 工作流终止_
```

## 🎯 三层异常推送原则确认

### 原则 1: Layer 1 数据获取异常

✅ **已实现**:
- 数据源输出格式验证
- 数据源数量验证
- 失败时抛出 `DataFetchException`
- 自动推送错误报告
- 工作流立即终止

**验证场景**:
- 百度 0 条 → failed
- 腾讯成功 → success
- 东方 0 条 → failed
- 成功数据源 1 个 < 要求 2 个 → **终止 + 推送**

---

### 原则 2: Layer 2 分析决策异常

✅ **已实现**:
- 输入数据格式验证（`AnalysisInput.validate()`）
- 输出数据格式验证（`AnalysisOutput.validate()`）
- 失败时抛出 `AnalysisException`
- 自动推送错误报告
- 工作流立即终止

**验证场景**:
- 输入 stocks 为空 → **验证失败 → 终止 + 推送**
- 输出缺少必填字段 → **验证失败 → 终止 + 推送**

---

### 原则 3: Layer 3 输出推送异常

✅ **已实现**:
- 输入数据格式验证（`PushInput.validate()`）
- 输出数据格式验证（`PushOutput.validate()`）
- 失败时抛出 `PushException`
- 自动推送错误报告（尝试二次推送）
- 工作流立即终止

**验证场景**:
- 输入 stocks 为空 → **验证失败 → 终止 + 推送**
- 推送 API 失败 → **记录错误 → 终止**
