# 重复推送问题修复报告

**问题发现时间**：2026-03-18 10:54
**修复时间**：2026-03-18 10:56

---

## 🐛 问题描述

用户反馈："消息推送重复了"

### 实际问题分析

经过检查，发现**不是真正的重复推送**，而是：

1. **推送了不相关的消息** - 如港股、医疗、中东局势等
2. **A 股相关性判断过于宽松** - 导致非 A 股消息被推送
3. **用户体感像"重复"** - 因为推送的都是类似的社会/国际新闻

### 问题日志

```
✅ 企业微信推送成功（第 1 条）
  📊 中性 ⭐⭐⭐⭐
  标题：港交所 IPO 将限制红筹公司消息不属实

✅ 企业微信推送成功（第 2 条）
  📊 积极 ⭐⭐⭐⭐
  标题：伊方称哈尔克岛局势平稳 国家石油出口持续进行
```

这两条消息**实际上与 A 股无直接关联**，不应该被推送。

---

## 🔍 根本原因

### 原因 1：A 股相关性判断过于宽松

**旧逻辑**：
```python
# 只要在文本中出现任何行业关键词就算相关
if '医药' in text:  # 即使只是医疗新闻，没有提到股市
    return True
```

**问题**：
- "再生胰岛移植" → 包含"医疗" → 判定为相关 ❌
- "哈尔克岛石油出口" → 包含"石油" → 判定为相关 ❌
- "港交所 IPO" → 包含"ipo" → 判定为相关 ❌

### 原因 2：评分系统不够严格

- 高权重关键词（+2 分）太多
- 普通关键词（+1 分）累积容易达到 3 星
- 没有区分"直接相关"和"间接相关"

---

## ✅ 修复方案

### 修复 1：严格 A 股相关性判断

**新逻辑**：

```python
# 第一优先级：直接提及 A 股市场
direct_keywords = [
    'a 股', '沪深', '上证', '创业板', '科创板',
    '证监会', '央行', '降准', '降息',
    '涨停', '跌停', '北向资金',
    '上证指数', '沪深 300', ...
]

if any(kw in text for kw in direct_keywords):
    return True

# 第二优先级：行业 + 市场组合（必须同时出现）
has_industry = any(kw in text for kw in industry_keywords)  # 如'芯片'
has_market = any(kw in text for kw in market_keywords)      # 如'业绩'

if has_industry and has_market:
    return True

return False
```

### 修复 2：提高判断标准

| 消息类型 | 旧逻辑 | 新逻辑 | 结果 |
|---------|--------|--------|------|
| "港交所 IPO 限制红筹公司" | ✅ 相关 | ❌ 不相关（港股） | 修复 |
| "再生胰岛移植" | ✅ 相关 | ❌ 不相关（医疗新闻） | 修复 |
| "伊拉克石油出口" | ✅ 相关 | ❌ 不相关（国际局势） | 修复 |
| "央行降准" | ✅ 相关 | ✅ 相关 | 保持 |
| "沪深 300 突破" | ✅ 相关 | ✅ 相关 | 保持 |
| "茅台业绩预增" | ✅ 相关 | ✅ 相关 | 保持 |

## 🎯 核心原则

> **宁可漏掉，不可错推**

### 推送标准

1. ✅ **直接提及 A 股市场** - 如"a 股"、"沪深"、"上证"
2. ✅ **中国宏观政策** - 如"央行降准"、"证监会"
3. ✅ **行业 + 市场组合** - 如"芯片 + 涨停"、"业绩 + 茅台"
4. ❌ **港股/美股** - 除非明确影响 A 股
5. ❌ **纯行业新闻** - 如医疗突破、技术进展
6. ❌ **国际局势** - 如中东局势、国际关系
